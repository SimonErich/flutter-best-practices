Follow Dart/Flutter
conventions

  • Conventions = Mutually agreed on
  • Code Style, Naming, Structure, ...
  • Defined by Dart Core Team
  • https://dart.dev/guides/language/effective-dart/style

Convention:
file and folder names

should always be snake_case_using_underscores

Convention:
classes, type definitions, enums, extensions

should always be UpperCamelCase

Convention:
variables, constants, class members, parameters

should always be lowerCamelCase

Convention:
Private variables and methods

should always be prefixed with _ (Undescore)
